- **ガイドライン**: 原稿編集 第2版 (`genkou-henshu-2`)

## 書籍根拠

D 節「a) 内容の点検」3 内容点検の具体的事項 (7)（p. 12）。書籍は「二様にとれる文章、修飾の関係が二様以上になる、どこに係るかわからない修飾など、誤読やわかりにくい文になっていないか」と点検項目として明示している。同 (8) では「一文に異なった事項を2つ盛り込んでいないか」を挙げ、「一般に一文に異なった事項を2つ盛り込むと、わかりにくい文章となる」と説明している。側注では本多勝一「日本語の作文技術」を引用し、「長い修飾語ほど先に、短いほど後に」という修飾順序の基準も参考情報として挙げている。

## 動作原理

**入力**: 一文（長文の場合は段落単位）  
**判断基準**:  
1. **修飾関係の多義性**: LLM が文を構文解析し、修飾語句が複数の被修飾語に係りうる（二重解釈可能な）箇所を検出する。  
2. **一文多事項**: 一文内に独立した事項が複数含まれており、文を分割した方が明瞭になると判断できる場合を検出する。  
3. **係り先の不明確さ**: 「しかし」「ただし」「また」などの接続表現の係り先が文脈から特定しにくい場合も対象とする。  
4. 以上に該当する場合、問題箇所の説明と、可能であれば修飾語句の位置を変えた代替案または文分割案を提示する。

**出力**: 多義・難解と判断された文、問題の種類（修飾多義 / 一文多事項 / 係り先不明確）の説明、可能な範囲での修正案。

## なぜ L1/L2 で不可能か

- 修飾語の係り先の多義性は文全体の構文ツリーを構築したうえで、複数の解釈が成立するかを判断する必要がある。形態素解析では依存関係を完全に解析することはできず、特に長い修飾節や複数の修飾語が絡む場合はさらに困難。
- 「一文に2つの事項が混在している」という判断は、文の内容が何について述べているかの意味理解が前提であり、パターンマッチでは検出不可能。
- 接続詞の係り先の不明確さは談話レベルの理解（前後の段落との関係）を必要とする。

## 例

### positive（指摘なし）

```
昨日、東京で開かれた大規模な国際会議が終了した。
```
（「東京で開かれた大規模な国際会議」という修飾関係は一義的に解釈できる）

```
実験は成功した。次のステップでは被験者数を増やす。
```
（2つの事項が2文に分かれており明確）

### negative（問題として指摘）

```
新しい教材を使って教えた経験のある教師が集まった。
```
（「新しい教材を使って」が「教えた」に掛かるのか「集まった」に掛かるのか二様に解釈できる。例: 「新しい教材を使って教えた経験のある教師が集まった」→「新しい教材を使った教授経験を持つ教師が集まった」など一義的な表現に修正することを提案）

```
研究チームは新手法を開発し、精度が向上した結果をまとめ、国際会議に投稿した。
```
（一文に「手法開発」「精度向上の確認」「論文投稿」の3事項が混在。文を分割することを提案）

## 想定される偽陽性・運用上の注意

- **文学的技巧**: 小説・詩において意図的な多義性・曖昧さは修辞的効果の一部である。novel モードでは適用しないことを強く推奨する。
- **短文**: 20字未満の短文は修飾関係の多義が生じにくく、このルールの対象外とする。
- severity は info（情報提供）に留める。「わかりにくい」の判断は主観的要素を含むため、著者・編集者の最終判断を尊重する。
- 提案する修正案は参考として提示するに留め、自動修正は行わない。
