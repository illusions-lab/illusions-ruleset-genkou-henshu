# geh-l3-hyoki-yure — 文書内の表記ゆれ（漢字/かな不統一）

> ⚠️ ステータス: L3（LLM支援）・未実装。SDK が L3 を支持した後に実装予定。本ドキュメントは仕様の先行定義です。

- **ruleId**: `geh-l3-hyoki-yure`
- **nameJa**: 文書内の表記ゆれ（漢字・かな不統一）検出
- **ガイドライン**: 原稿編集 第2版 (`genkou-henshu-2`)
- **レベル**: L3（LLM支援）
- **想定 severity**: info
- **適用モード案**: official / academic / blog / novel

## 書籍根拠

E 節「a) 表記の整理の原則」1 表記の整理、2 表記の扱い。書籍は一般書において著者の立場を尊重しながら表記をできる限り整理・統一することを求め、整理方針のメモや用語リストの作成という具体的な作業を示している。同節 b) 6「仮名書きが望ましい語」では代名詞・副詞・接続詞等について、どの範囲で仮名書きとするか一定の方針を決めることを求めている。

## 動作原理

**入力**: 文書全体のテキスト  
**判断基準**:  
1. LLM が文書全体を走査し、同一の語が異なる表記（漢字表記・かな表記・混在）で現れている箇所を列挙する。  
2. 対象語の候補は（a）接続詞・副詞・代名詞等の機能語（例: 「ため」/「為」、「とき」/「時」、「もの」/「物」）と（b）専門用語・片仮名語（例: 「コンピューター」/「コンピュータ」）の両方を含む。  
3. 同一語義であることを文脈から確認したうえで表記が異なる場合に指摘する。固有名詞や意図的な振り仮名付与は除外する。  
4. 文書内で多数派の表記を「基準表記」として提示し、少数派の箇所を指摘する。

**出力**: 表記ゆれが検出された語とその全出現箇所のリスト、多数派表記の提示。

## なぜ L1/L2 で不可能か

- 「時」と「とき」の使い分けは、一般名詞（「時間」の意）か機能語（「〜する場合」の意）かで異なり、形態素解析で品詞は取れても語義の区別（名詞用法か接続用法か）は文脈判断が必要。
- 「もの」/「物」、「こと」/「事」なども同様で、抽象・機能用法か具体名詞かを L1/L2 では判定できない。
- 片仮名語の揺れ（末尾長音の有無以外のバリエーション）は固定パターンが無数にあり、辞書を用いた L1 で全対応するのは現実的でない。
- 文書全体を横断して「同一語の複数表記」を検出することは、文書レベルの統計処理とセマンティクス確認を組み合わせる必要があり、局所パターンマッチのみでは不可能。

## 例

### positive（指摘なし）

```
このことは重要だ。そのことを忘れてはならない。次のことを確認する。
```
（「こと」が全箇所でかな表記に統一されている）

### negative（表記ゆれとして指摘）

```
このことは重要だ。その事を忘れてはならない。次のことを確認する。
```
（「こと」と「事」が混在。多数派「こと」に統一するよう提案）

```
データをコンピューターで処理する。コンピュータの性能が向上した。
```
（「コンピューター」と「コンピュータ」が混在。いずれかに統一するよう提案）

## 想定される偽陽性・運用上の注意

- **意図的な表記の使い分け**: 著者が「時間」としての「時」と接続詞的「とき」を意識的に使い分けている場合は正用。LLM は文脈を確認し、同一語義であるかを判断すること。
- **固有名詞**: 固有名詞の揺れ（組織名の正式表記と略称等）は本ルールの対象外。
- **引用文**: 引用文内の表記は原文に従うため除外する。
- severity は info（情報提供）に留める。表記の最終決定は著者・編集者の方針によるため、機械的な自動修正は不適切。
- 文書が短い場合（500字未満等）は「多数派」の判定が不安定になる。文書長に応じた閾値設定を検討する。
