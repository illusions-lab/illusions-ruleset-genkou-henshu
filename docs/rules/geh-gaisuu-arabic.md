# geh-gaisuu-arabic — 概数のアラビア数字表記検出

- **ruleId**: `geh-gaisuu-arabic`
- **ガイドライン**: 原稿編集 第2版 (`genkou-henshu-2`)
- **レベル**: L1（正規表現）
- **既定**: 有効 / severity = warning
- **適用モード**: 全モード（novel / official / blog / academic / sns）

## 意図

原稿編集 第2版 E 節 15 項「横組の数字表記 (7) 不確定数の表記」は、
概数（ばくぜんとした数・不確定数）をアラビア数字と概数語の組み合わせで書いてはならないことを定めており、
「数十円」「百余人」のように漢数字で統一するのが原則とされている。
また、「500余り」のような後置用法は許容とされる。

## 出典

原稿編集 第2版（日本エディタースクール）E 表記の整理 b) 表記整理の具体的事項 15 横組の数字表記 (7) 不確定数の表記

## 例

| 区分 | テキスト | 期待 |
| ---- | -------- | ---- |
| 正例 | `数十名の参加者が集まった。` | 指摘なし |
| 正例 | `百余人が集会に参加した。` | 指摘なし |
| 誤例 | `数10名の参加者が集まった。` | 警告（漢数字化を提案） |
| 誤例 | `何10回も試みた。` | 警告（漢数字化を提案） |
| 誤例 | `100余人が参加した。` | 警告（漢数字化を提案） |

## 偽陽性回避

- 検出するのは「数+アラビア数字」「何+アラビア数字」「アラビア数字+余（り以外）」の3パターン。
- 「500余り（の）」（「余り」と続く場合）は許容表現のため除外（原文: 「500円余りはよい」）。
- 修正候補は自動生成が困難なため、警告のみとして手動修正を促す（replacement は原文を返す）。

## 偽陽性リスク

**低**: パターンが明確。「数10行のコード」「何10倍」等、すべて概数語との組み合わせのみが対象。
「500余りの在庫」のような正用表現は「余り」の先読みで除外済み。
