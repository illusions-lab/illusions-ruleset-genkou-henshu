# geh-bracket-mismatch — かぎ括弧の対応ズレ検出

- **ruleId**: `geh-bracket-mismatch`
- **ガイドライン**: 原稿編集 第2版 (`genkou-henshu-2`)
- **レベル**: L1（正規表現）
- **既定**: 有効 / severity = warning
- **適用モード**: 全モード（novel / official / blog / academic / sns）

## 意図

原稿編集 第2版 E 節 19 項「約物の使い方 ② 括弧類の使用法」は、
括弧類の起こし（開き括弧）と受け（閉じ括弧）が必ず対応していることを確認するよう求めている。

かぎ括弧「」の開き（「）と閉じ（」）が一致しない場合に検出し、閉じ忘れ・開き忘れを警告する。

## 出典

原稿編集 第2版（日本エディタースクール）E 表記の整理 b) 表記整理の具体的事項 19 約物の使い方 ② 括弧類の使用法

## 例

| 区分 | テキスト | 期待 |
| ---- | -------- | ---- |
| 正例 | `彼は「今日はいい天気だ」と言った。` | 指摘なし |
| 正例 | `「私は『吾輩は猫である』を読んだ」と彼は言った。` | 指摘なし（入れ子もカウント合計で判定） |
| 誤例 | `彼は「今日はいい天気だと言った。` | `「` の閉じ忘れとして警告 |
| 誤例 | `彼は今日はいい天気だ」と言った。` | 対応する `「` のない `」` として警告 |

## 偽陽性回避

- 入力テキスト全体で開き・閉じのカウントを照合する方式。
- ページをまたぐ会話（前段落で開き、次段落で閉じる）は偽陽性になる可能性がある。
  その場合は段落ごとに入力を分割するか、本ルールを手動で確認するとよい。

## 偽陽性リスク

**低〜中**: 同一入力内での括弧対応は高精度で検出できる。
ただし複数段落にまたがる長い会話文では偽陽性が生じる可能性がある（中リスク）。
