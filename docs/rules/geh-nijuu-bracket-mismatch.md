# geh-nijuu-bracket-mismatch — 二重かぎ括弧の対応ズレ検出

- **ruleId**: `geh-nijuu-bracket-mismatch`
- **ガイドライン**: 原稿編集 第2版 (`genkou-henshu-2`)
- **レベル**: L1（正規表現）
- **既定**: 有効 / severity = warning
- **適用モード**: 全モード（novel / official / blog / academic / sns）

## 意図

原稿編集 第2版 E 節 19 項「約物の使い方」では、二重かぎ（『』）は書名・雑誌名を
くくる場合や、一重かぎの中で用いる場合に使うとし、括弧類の対応確認を必須としている。

二重かぎ括弧「』」の開き（『）と閉じ（』）が一致しない場合を検出する。

## 出典

原稿編集 第2版（日本エディタースクール）E 表記の整理 b) 表記整理の具体的事項 19 約物の使い方 ② 括弧類の使用法

## 例

| 区分 | テキスト | 期待 |
| ---- | -------- | ---- |
| 正例 | `『吾輩は猫である』を読んだ。` | 指摘なし |
| 誤例 | `『吾輩は猫であるを読んだ。` | `『` の閉じ忘れとして警告 |
| 誤例 | `吾輩は猫である』を読んだ。` | 対応する `『` のない `』` として警告 |

## 偽陽性回避

- 入力テキスト全体で開き・閉じのカウントを照合する方式。
- geh-bracket-mismatch と同様、ページをまたぐ表現では偽陽性の可能性がある。

## 偽陽性リスク

**低**: 二重かぎ括弧は書名・引用の固定用途で使われるため、段落をまたぐ頻度は低く、
一般的な原稿では高精度で対応ズレを検出できる。
